Relation:getRelationHistory It returns the relation history of a card starting from a "Relation" object in which only "Class1Name" and "Card1Id" were defined.

Parameters:

  • domain_name - Domain used for the relation. (string)
  • class1name - ClassName of the first card taking part in the relation (string)
  • card1id - Identifier of the first card which takes part in the relation (number)
  • class2name - ClassName of the second card which takes part in the relation. (string)
  • card2id - Identifier of the second card which takes part in the relation. (number)
  • status - Relation status ('A' = active, 'N' = removed) (string)
  • begin_date - Date when the relation was created (format YYYY-MM-DDThh:mm:ssZ) (date)
  • end_date - Date when the relation was created (format YYYY-MM-DDThh:mm:ssZ) (date)

Returns:

    table
